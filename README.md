# Lili Miller — портфолио авторских кукол

Адаптивный сайт-портфолио художницы Lili Miller в визуальном направлении
«Фарфор в полумраке». Проект знакомит с коллекциями авторских фарфоровых кукол,
историями персонажей, выставками и творческим процессом.

## Возможности

- адаптивная вёрстка для мобильных и настольных экранов;
- переключение русского и английского языков;
- интерактивные карточки работ и мобильное меню;
- анимации появления секций;
- каталог изображений и подписей из Telegram-источников.

## Просмотр

Для просмотра достаточно открыть `index.html` в браузере. Чтобы запустить сайт через
локальный HTTP-сервер, выполните из корня проекта:

```bash
python3 -m http.server 4173
```

После этого откройте [http://localhost:4173](http://localhost:4173).

## Структура

- `index.html` — главная страница прототипа;
- `styles.css` — адаптивная дизайн-система и анимации;
- `script.js` — мобильное меню, появление секций, карточки работ и RU/EN;
- `data/works.json` — структурированные данные о работах;
- `works/*/index.html` — индексируемые страницы работ с индивидуальными SEO-метаданными;
- `sitemap.xml` и `robots.txt` — карта сайта и правила обхода поисковыми роботами;
- `assets/images/social-preview.jpg` — превью 1200×630 для Open Graph и X Cards;
- `content/content-inventory.md` — черновой реестр коллекций и работ;
- `content/telegram-media-catalog.md` — каталог фотографий с подписями Telegram-постов;
- `research/benchmark-and-design-directions.md` — исследование референсов;
- `assets/images/telegram-originals/` — локальный архив исходников, исключённый из Git;
- `assets/images/telegram-web/` — оптимизированные WebP-изображения для сайта;
- `data/telegram-media-manifest.json` — машиночитаемый индекс web-медиа;
- `tools/export_telegram_images.py` — повторяемая выгрузка с прогрессбаром.
- `tools/optimize_telegram_images.py` — WebP-конвертация с ограничением размера и прогрессбаром.
- `tools/generate_seo_pages.py` — генерация страниц работ и sitemap из `data/works.json`.

## Технологии

Сайт работает без сборщика и внешних JavaScript-зависимостей: используются HTML5,
CSS3 и нативный JavaScript. Для служебных скриптов требуется Python 3.

## Выгрузка изображений из Telegram

Экспортёр получает все фотографии и графические документы из:

- `Douls for soul site` (`-5265896039`);
- `@lilimillerdoll` (`-1001669754520`).

Запуск через окружение Telegram MCP:

```bash
/opt/homebrew/bin/uv \
  --directory /Users/legonaut/Projects/infra/telegram-mcp \
  run python /Users/legonaut/Projects/lilimiller/tools/export_telegram_images.py
```

Во время работы отображается общий прогресс, количество новых файлов, локальный кэш и ошибки. Повторный запуск не скачивает существующие файлы заново, но обновляет метаданные и контрольные суммы.

Поле `post_comment` в `data/telegram-media-manifest.json` содержит подпись к Telegram-посту. Для альбомов одна подпись автоматически связывается со всеми фотографиями. Для публичного канала также сохраняется `post_url`; пустая подпись обозначается в Markdown-каталоге символом `—`.

После выгрузки подготовьте web-версии изображений:

```bash
python3 tools/optimize_telegram_images.py
```

По умолчанию изображения конвертируются в WebP с качеством 82, длинной стороной не более 2000 px и без EXIF-метаданных. Оригиналы остаются локально и не попадают в историю Git.

После изменения `data/works.json` обновите индексируемые страницы и карту сайта:

```bash
python3 tools/generate_seo_pages.py
```
